Three files were rewritten to eliminate the single global `SessionState` that caused all users to share state:
- `backend/orchestrator/session.py` — `LiveSessionManager` now uses `Dict[str, ...]` keyed by `session_id`. All methods (`register_connection`, `disconnect`, `set_personas`, `get_personas`, `get_state`, `add_transcript_chunk`, `update_agent_state`, `broadcast_state`) take `session_id`.
- `backend/orchestrator/loop.py` — `OrchestratorLoop` uses per-session `asyncio.Lock` via `_locks: Dict[str, asyncio.Lock]`. All methods take `session_id`.
- `backend/api/routers/live.py` — WebSocket accepts first, then waits for `init_session` message to learn `session_id`. `orchestrator = OrchestratorLoop(session_manager)` module-level singleton.
